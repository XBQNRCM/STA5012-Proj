## FAVOR+ 在近似什么

注意力里常用的核是（在你这边的记号下）  
\(K(q,k)=\exp(q^\top k/\sqrt{d})\)。

先把 \(x\) 换成scaled变量  
\(z = x / d^{1/4}\)，  
则  
\(z_q^\top z_k = (q^\top k)/\sqrt{d}\)，  
所以要近似的是 \(\exp(z_q^\top z_k)\)。

FAVOR+ 用一组随机方向 \(w_i\sim\mathcal N(0,I)\)，构造**正**特征  
\[
\phi_i(z)=\frac{1}{\sqrt{m}}\exp\!\Big(w_i^\top z-\frac{\|z\|^2}{2}\Big).
\]  
对 \(w\) 取期望时，有  
\(\mathbb E_w[\phi(z_q)^\top \phi(z_k)]=\exp(z_q^\top z_k)\)，  
也就是 Monte Carlo 用有限个 \(m\) 条 \(w_i\) 去估计这个核。

代码里和公式一一对应：

- `z = x * (dim_d ** -0.25)` → \(z=x/d^{1/4}\)
- `log_h = z @ omega.T - (z * z).sum(...) / 2` → 每个 \(i\) 上是 \(w_i^\top z - \|z\|^2/2\)
- `exp(log_h) / sqrt(dim_m)` → 除以 \(\sqrt{m}\)

所以这是标准的 **Performer / FAVOR+ 正特征** 形式，不是随便的 MLP。

## 「嵌套截断」指什么

你要扫一堆特征维 \(m\in\{m_1,\ldots,m_K\}\)。若 **每个 \(m\) 都独立重采一整张 \(\omega\in\mathbb R^{m\times d}\)**，则不同 \(m\) 对应的是**完全不同的**随机特征集合，误差随 \(m\) 的变化会夹杂「换了一套随机数」的噪声，单条 trial 下 **RelErr 随 \(m\) 未必单调**。

**嵌套截断**的做法是：在同一个 `(map, trial)` 里只采一次大矩阵  
\(\omega_{\mathrm{full}}\in\mathbb R^{M_{\max}\times d}\)，  
对具体的 \(m\) 只用其**前 \(m\) 行**：  
\(\omega_m = \omega_{\mathrm{full}}[:m,:]\)。

这样：

- \(m\) 从小到大时，特征向量是**在同一条随机特征族里不断加长前缀**，而不是换一批独立的 \(w\)。
- 在同一 trial 内，增大 \(m\) 只会**增加**更多 Monte Carlo 样本，期望上误差应随 \(m\) 变好，曲线更「符合直觉」、更可比较。

你 docstring 里写的「先采 `omega_full[max_m, d]`，再取 `omega_full[:m]`」就是这个意思；**类本身**只接受已经裁好的 `omega`；**嵌套**是在 `runner` 里对每个 trial 采 `omega_full` 再切片传入（类实现保持「给定 \(\omega\) 就算 \(\phi\)」）。

## 小结

| 概念 | 在代码中的体现 |
|------|----------------|
| FAVOR+ 正特征 | `exp(w_i^T z - \|z\|^2/2)/sqrt(m)` |
| 与 \(\exp(q^\top k/\sqrt{d})\) 对齐 | \(z=x/d^{1/4}\) |
| 嵌套截断 | 同一 trial 共用 \(\omega_{\mathrm{full}}\) 的前 \(m\) 行，而不是每个 \(m\) 独立重采 \(\omega\) |
